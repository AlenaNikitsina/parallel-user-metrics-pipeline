# ETL-пакет для агрегации пользовательских метрик (Airflow + ClickHouse)

Данный проект представляет собой ETL-пакет, разработанный в Apache Airflow. Основная задача — ежедневный расчет поведенческих метрик пользователей на основе двух источников данных (`feed_actions` и `message_actions`), их последующее объединение и детализация по ключевым срезам (пол, возраст, ОС) с записью результатов в ClickHouse.

## 📌 Цели проекта

1.  **Автоматизация расчетов:** Создание надежного DAG, который автоматически запускается каждый день и обрабатывает данные за предыдущий день (вчера).
2.  **Параллельная обработка:** Независимый расчет базовых метрик из двух разных источников для оптимизации времени выполнения.
3.  **Детальный анализ:** Расчет агрегированных метрик в разрезе демографических и технологических характеристик пользователей (gender, age, os).
4.  **Накопление данных:** Сохранение результатов в целевой таблице ClickHouse с поддержкой ежедневного добавления (инкрементальная загрузка).

## 📁 Описание набора данных

Проект работает с двумя входящими таблицами, которые символизируют разные типы активности пользователей:

*   **`feed_actions`** (Лента событий):
    *   Содержит данные о взаимодействии пользователя с контентом в ленте.
    *   **Метрики:** Количество просмотров и количество поставленных лайков.

*   **`message_actions`** (Действия с сообщениями):
    *   Содержит данные об обмене личными сообщениями.
    *   **Метрики:** Количество отправленных сообщений, полученных сообщений, количество уникальных получателей (users_sent) и количество уникальных отправителей (users_received).

## 🏗️ Основные шаги DAG

DAG состоит из нескольких логических блоков, каждый из которых реализован в виде отдельного таска:

1.  **Параллельный расчет (`get_feed_actions` и `get_message_actions`):**
    *   Два параллельных таска читают исходные таблицы и агрегируют данные на уровне **пользователя** (`user_id`) за вчерашний день.
    *   Формируются две временные витрины: одна с полями `user_id, views, likes`, вторая с `user_id, messages_sent, messages_received, users_sent, users_received`.

2.  **Объединение данных (`join_tasks`):**
    *   Таск ожидает завершения двух предыдущих (через `TriggerRule.ALL_SUCCESS`).
    *   Производит полное внешнее соединение (`FULL OUTER JOIN`) двух датафреймов по `user_id`, объединяя все метрики в одну плоскую таблицу на уровне пользователя.

3.  **Обогащение и расчет срезов:**
    *   К объединенным данным подтягиваются справочные данные о пользователях: пол (`gender`), возраст (`age`) и операционная система (`os`).
    *   Создаются три параллельных таска для расчета итоговых агрегатов:
        *   **`agg_by_gender`:** Группировка метрик по полу.
        *   **`agg_by_age`:** Группировка метрик по возрастным группам.
        *   **`agg_by_os`:** Группировка метрик по операционной системе.

4.  **Фиксация результатов (`union_and_save`):**
    *   Результаты трех агрегирующих тасков объединяются в один финальный датасет.
    *   Датасет приводится к целевой структуре (см. ниже).
    *   Происходит запись (добавление/вставка) в целевую таблицу ClickHouse.

## 📊 Структура финальной таблицы в ClickHouse

| Имя столбца          | Тип данных (ClickHouse) | Описание                                                       |
| :------------------- | :----------------------- | :------------------------------------------------------------- |
| `event_date`         | `Date`                   | Дата, за которую рассчитаны метрики.                           |
| `dimension`          | `String`                 | Название среза (например, 'gender', 'age', 'os').             |
| `dimension_value`    | `String`                 | Значение среза (например, 'male', '18-25', 'iOS').            |
| `views`              | `UInt64`                 | Суммарное число просмотров контента в срезе.                   |
| `likes`              | `UInt64`                 | Суммарное число лайков контента в срезе.                       |
| `messages_received`  | `UInt64`                 | Суммарное число полученных сообщений.                          |
| `messages_sent`      | `UInt64`                 | Суммарное число отправленных сообщений.                        |
| `users_received`     | `UInt64`                 | Скольким уникальным пользователям писали (получатели).         |
| `users_sent`         | `UInt64`                 | От скольких уникальных пользователей получили сообщения (отправители). |

## 🛠️ Используемые технологии и библиотеки

*   **Оркестрация:** Apache Airflow
*   **Обработка данных:** Python (библиотеки: `pandas`, `numpy`)
*   **Хранилище данных:** ClickHouse
*   **Коннекторы:** `clickhouse-driver` / `apache-airflow-providers-clickhouse`
*   **Взаимодействие с данными (условно):** SQL (для извлечения данных из источников)

## 🚀 Результат

В результате работы DAG в ClickHouse ежедневно появляется порция данных, которая позволяет аналитикам и дашбордам отвечать на вопросы:

*   "Как отличается потребление контента (лайки, просмотры) у пользователей Android и iOS?"
*   "Люди какого возраста отправляют больше всего сообщений?"
*   "Кто более активен в ленте: мужчины или женщины?"

Данные хранятся в удобном для аналитики виде (в разрезах), что упрощает построение отчетов и дальнейший анализ.
